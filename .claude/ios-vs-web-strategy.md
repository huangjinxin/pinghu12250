# iOS App vs Web 功能策略分析

> 生成日期: 2026-02-22 | 用途: 功能规划参考，避免重复分析

## 当前状态

- **iOS App**: 183个Swift文件，SwiftUI + MVVM，覆盖核心功能
- **Web端**: 70+页面，Vue3，功能最全面（含管理后台、社交、游戏等）
- **共用后端**: Express + Prisma + PostgreSQL，API已统一

---

## 优先做到 iOS App 的功能

### 1. 手写/书法 — 最高优先级
- PencilKit + OCR 基础已有，Apple Pencil 压感/倾斜检测
- 写字评测(writingEvaluation)在iPad上才是真正使用场景
- Web端书法只能"看"，App端才能"练"

### 2. 语音朗诵/录音
- AudioService + VoiceInputService 已有
- 诗词朗诵录制、语音日记，原生音频API延迟低质量高
- Web端 MediaRecorder 兼容性和体验差

### 3. 离线阅读 + 笔记同步
- CoreData + SyncService local-first架构已有
- 无网环境（课堂/通勤）仍可用
- Web离线能力(PWA)远不如原生可靠

### 4. 专注模式/学习计时器
- 系统级Focus Mode、Screen Time API
- 后台计时不因切Tab中断
- 本地通知提醒休息/继续

### 5. 日记 + 拍照
- 拍照→OCR→AI分析链路更流畅
- 相册/相机直接调用
- Widget桌面提醒"今天还没写日记"

### 6. 家长端监控
- 推送通知（完成作业、获得奖励）
- 快速查看无需浏览器登录
- 轻量级"家长看板"

---

## 留在 Web 的功能

### 1. 管理后台(Admin) — 明确留Web
- 用户/班级管理、数据分析、审核系统
- 大屏幕多标签页批量操作
- 用户量小（仅老师/管理员）

### 2. 作品编辑器(WorkEditor)
- HTML作品需富文本编辑，Web编辑器生态更成熟
- 复杂排版触屏操作不便

### 3. 游戏大厅/H5游戏
- Web技术实现，移植无意义
- App内用WKWebView嵌入即可

### 4. 商城/支付码管理
- 低频操作Web够用
- iOS支付涉及App Store IAP审核问题

### 5. 数据统计/排行榜详情
- 复杂图表Web用ECharts更灵活
- App端展示摘要，详情跳Web

---

## 两端都需要但侧重不同

| 功能 | Web侧重 | iOS侧重 |
|------|----------|----------|
| 教材阅读 | 管理上传、公开浏览 | 深度阅读、批注、手写 |
| AI对话 | 完整对话历史管理 | 即时交互、语音输入 |
| 日记 | 浏览、AI分析报告 | 写作、拍照、语音 |
| 作品展示 | 完整画廊、评论互动 | 快速浏览、点赞分享 |
| 积分/钱包 | 详细流水、兑换商城 | 余额查看、扫码支付 |

## 技术路线

### 短期
- 打磨iOS手写/书法模块到可用状态（最大差异化）
- 完善离线阅读+笔记同步可靠性
- 加推送通知（作业提醒、奖励通知、家长消息）

### 中期
- iOS Widget（学习进度、今日任务、日记提醒）
- Shortcuts/Siri集成（"帮我打开语文课本"）
- iPad分屏：左教材右笔记

### 长期
- 评估iPhone版需求（当前iPad教育场景为主）
- Apple Watch学习提醒/计时
- SharePlay协作学习

### 架构
- 后端API共用，方向正确
- 统一API版本管理避免两端不一致
- 考虑WebSocket替代轮询（消息/通知），两端受益
