# 苹湖少儿空间 - 项目深度分析报告

> 更新时间: 2026-02-01
> 分析范围: Web端 (frontend + backend)

## 一、项目概述

**项目名称**: 苹湖少儿空间 (pinghu12250)
**项目定位**: 儿童学习成长管理平台
**核心价值**: 为学生、家长、教师提供全方位的儿童成长记录和学习管理平台

### 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Vue 3 + Vite |
| UI组件库 | Naive UI + TailwindCSS |
| 状态管理 | Pinia |
| 后端框架 | Express.js |
| ORM | Prisma |
| 数据库 | PostgreSQL 15 (含 pgvector) |
| 容器化 | Docker + Docker Compose |

### 端口配置

- 前端: 12250
- 后端: 12251
- 数据库: 12252

## 二、功能模块总览

### 2.1 用户系统
- **角色**: STUDENT(学生) / PARENT(家长) / TEACHER(老师) / ADMIN(管理员)
- **功能**: 注册登录、JWT认证、2FA两步验证、家长-学生关联、头像管理

### 2.2 内容创作系统

| 模块 | 功能描述 | 特色功能 |
|------|----------|----------|
| 日记 | 富文本日记创作 | 心情天气标记、AI分析、查重检测 |
| HTML作品 | 在线代码编辑器 | 实时预览、Fork、市集交易 |
| 诗歌作品 | 诗歌创作发布 | 封面自动生成、公开展示 |
| 创意作品 | 书法/摄影作品 | 分类管理、审核流程 |
| 笔记 | 课堂/读书/教材笔记 | Tiptap富文本、科目分类 |

### 2.3 电子课本系统 ⭐核心模块

这是项目最复杂的模块，集成了多项创新功能：

**PDF阅读器**:
- 单页/翻书模式切换
- 文本选择与标注
- 区域截图
- 触摸选择模式(移动端优化)

**AI辅助学习**:
- 三层提示词架构(系统级→科目级→教材级)
- 多AI服务商支持(OpenAI兼容API)
- 流式对话(SSE)
- 文本解释、问题解答、练习题生成

**笔记系统**:
- 页面级笔记
- 查字功能(集成字典API)
- 专注模式(FocusMode)

### 2.4 积分与激励系统

| 模块 | 功能描述 |
|------|----------|
| 积分系统 | 每日积分上限、多种积分来源、积分兑换虎币 |
| 成就系统 | 计数型/连续型/里程碑型成就、进度追踪 |
| 每日挑战 | 自动生成挑战、完成领取奖励 |

### 2.5 社交系统

- **关注系统**: 单向关注、双向好友、推荐
- **动态系统**: 发布动态、双时间轴、点赞评论
- **知识问答**: 提问回答、悬赏积分、采纳答案
- **实时消息**: 私信、已读状态、Socket.io推送

### 2.6 媒体库系统

| 模块 | 功能 |
|------|------|
| 书籍 | 书架管理、阅读状态、读后感 |
| 音乐 | 音乐库、听歌日志 |
| 影视 | 影视库、观影日志 |
| 游戏 | 游戏库、评分评测、RAWG API集成 |

### 2.7 学习追踪系统

- **学习计时**: 项目管理、计时器、热力图统计
- **作业管理**: 作业记录、科目分类、统计

### 2.8 班级与任务系统

- **班级管理**: 学校/班级/教师-学生关联
- **任务系统**: 老师发布、学生提交、评分

### 2.9 钱包与支付系统

- **虚拟钱包**: 虎币、交易记录、积分兑换
- **市集系统**: 付费内容发布与购买
- **支付码**: 扫码支付、订单管理

### 2.10 管理后台

用户管理、班级管理、校区管理、游戏管理、奖罚管理、支付码管理、活动日志、数据看板、反馈管理、分析统计

## 三、解决的实际问题

### 3.1 学习激励问题
**问题**: 儿童学习缺乏动力和持续性
**解决方案**:
- 积分系统激励日常学习行为
- 成就系统提供长期目标
- 每日挑战保持参与度
- 排行榜激发竞争意识

### 3.2 学习记录问题
**问题**: 学习过程难以追踪和量化
**解决方案**:
- 学习计时器精确记录学习时长
- 热力图可视化学习频率
- 日记系统记录学习心得
- 笔记系统整理知识点

### 3.3 家校沟通问题
**问题**: 家长难以了解孩子学习情况
**解决方案**:
- 家长账号关联查看孩子数据
- 任务系统透明化作业情况
- 奖罚记录可追溯
- 成长时间轴展示进步

### 3.4 电子课本学习问题
**问题**: 传统PDF阅读缺乏互动性
**解决方案**:
- AI辅助解释生僻字词
- 智能问答解决学习疑问
- 自动生成练习题巩固知识
- 笔记系统关联课本内容

### 3.5 创作展示问题
**问题**: 儿童创作缺乏展示平台
**解决方案**:
- HTML作品在线编辑和展示
- 诗歌作品公开发布
- 市集系统支持作品交易
- Fork功能促进学习交流

## 五、开发注意事项

### 5.1 必须遵守的规则

```javascript
// Prisma 必须使用单例
const prisma = require('../lib/prisma');  // ✓
const prisma = new PrismaClient();        // ✗

// API响应格式
{ success: true, data: {...} }   // 成功
{ success: false, error: '...' } // 失败

// 前端API文件
frontend/src/api/index.js  // 唯一API文件，不要新建
```

### 5.2 层级调用规则

```
view      → [component, composable, store, utils]
component → [composable, store, utils] ✗api
composable→ [api, store, utils]
controller→ [service] ✗prisma直接调用
```

### 5.3 命名规范

- 组件: `[Module][Name].vue` (如 TextbookReader.vue)
- Composable: `use[Module][Action].js` (如 useTextbookChat.js)
- Service: `[module]Service.js` (如 textbookService.js)
- API对象: `[module]API` (如 textbookAPI)
